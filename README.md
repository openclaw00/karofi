# karofi

Static storefront for Karofi-style product pages with a Vercel order API.

## Run locally

Use the bundled Node in this Codex workspace:

```sh
/Users/minhang/.cache/codex-runtimes/codex-primary-runtime/dependencies/node/bin/node server.js
```

Open `http://localhost:4173`.

## Orders

Checkout posts orders to `POST /api/orders`.

On Vercel, `api/orders.js` does two things:

- Saves the order to Supabase.
- Sends an email notification through Resend.

Supabase is the source of truth. If the email notification fails, checkout still succeeds and the confirmation page warns that the saved order should be checked in the database.

## Supabase setup

1. Create a Supabase project.
2. Open the SQL editor and run:

```sql
create table if not exists public.orders (
  id bigint generated by default as identity primary key,
  order_id text not null unique,
  created_at timestamptz not null default now(),
  status text not null default 'Chờ xử lý',
  customer jsonb not null,
  items jsonb not null,
  total numeric not null,
  raw_order jsonb not null
);

alter table public.orders enable row level security;

create index if not exists orders_created_at_idx on public.orders (created_at desc);
```

3. In Supabase Project Settings, copy the Project URL and service role key.
4. Add these environment variables in Vercel:

```sh
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
SUPABASE_ORDERS_TABLE=orders
```

Do not put `SUPABASE_SERVICE_ROLE_KEY` in browser code. It belongs only in Vercel environment variables.

## Email setup

1. Create a Resend account.
2. Verify your sending domain, or use Resend's test sender while developing.
3. Add these environment variables in Vercel:

```sh
RESEND_API_KEY=re_xxxxxxxxx
ORDER_EMAIL_TO=you@example.com
ORDER_EMAIL_FROM=Karofi Store <orders@yourdomain.com>
```

## Fulfillment dashboard

The dashboard at `orders.html` reads from `GET /api/orders`, which reads Supabase through the Vercel API. On Vercel, protect it with a long random token:

```sh
ADMIN_ORDERS_TOKEN=change-this-long-random-admin-password
```

When `ADMIN_ORDERS_TOKEN` is set, `orders.html` prompts for the token before showing customer details. The API requires this token for deployed dashboard reads. Keep this page out of public navigation; it is for internal fulfillment only.

## Vercel deployment checklist

1. Import this folder into Vercel as a static project with Node serverless functions.
2. Add all Supabase, Resend, and admin environment variables in Vercel Project Settings.
3. Redeploy after adding or changing env vars.
4. Submit a real test order online.
5. Confirm the row appears in Supabase and the notification email arrives.

## Google Sheets fallback

The API still supports Google Sheets if `SUPABASE_URL` and `SUPABASE_SERVICE_ROLE_KEY` are not set. Supabase is simpler and is the recommended setup.

Never put Supabase service role keys, Google private keys, or Resend API keys in browser code. They belong only in Vercel environment variables.
